# Чеклист аудита контекста проекта

## Быстрая проверка

- Есть ли `CURRENT_STATE.md`?
- Есть ли `NEXT_WORK.md`?
- Есть ли `PROJECT_MAP.md`?
- Понятно ли, какие папки являются архивом?
- Есть ли большие монолитные файлы?
- Есть ли дубли документации?
- Есть ли generated/dependency папки, которые ИИ не должен читать?
- Есть ли команда запуска?
- Есть ли минимальная команда проверки?

## Красные флаги

- Пользователь просит "прочитай весь проект" для маленькой задачи.
- В одном чате смешаны баги, дизайн, стратегия и исследование.
- Нет короткой фиксации текущего состояния.
- Большой файл читается целиком вместо поиска по функциям.
- Архивные документы используются как актуальная спецификация.

## Рекомендация

Перед новой работой создать или обновить:

```text
00_Context/CURRENT_STATE.md
00_Context/NEXT_WORK.md
00_Context/PROJECT_MAP.md
```

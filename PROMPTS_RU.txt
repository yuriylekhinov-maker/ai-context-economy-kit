AI Context Economy Kit
5 базовых промптов для экономной работы с ИИ

============================================================
1. START EXISTING PROJECT
============================================================

Используй режим разумной экономии контекста.

Проект: <путь к проекту>

Сначала найди минимальный контекст для входа в проект:
- README;
- context/current-state/brief файлы, если они есть;
- карту проекта или структуру папок.

Не читай весь проект целиком.
Не читай архивы, большие логи, изображения, ассеты и generated/dependency папки без явной причины.
Большие файлы сначала исследуй через поиск по функциям, ошибкам, id, текстам интерфейса или ключевым словам.

Моя задача:
<коротко описать задачу>

Перед изменениями кратко скажи, какие файлы собираешься трогать.
После работы ответь коротко: что изменено, где, чем проверено, какие риски остались.

============================================================
2. FIX BUG CHEAP
============================================================

Нужно исправить баг экономно по контексту.

Проект: <путь к проекту>

Симптом:
<что происходит>

Ожидаемое поведение:
<что должно происходить>

Команда/действие, которое воспроизводит проблему:
<команда, экран, клик, сценарий>

Ограничения:
- не читать весь проект;
- сначала найти релевантные места через поиск;
- не делать широкий рефакторинг;
- менять только минимально нужные файлы;
- запускать только релевантные проверки.

Если мой запрос можно сделать дешевле, сначала напиши: "Можно дешевле:" и предложи более точный вариант.

============================================================
3. REVIEW UI CHEAP
============================================================

Проверь UI экономно по контексту.

Проект: <путь к проекту>

Экран/область:
<например: профиль, мобильная ширина, форма логина, главный экран>

Что проверить:
- наложение текста;
- читаемость;
- адаптивность;
- кликабельность;
- очевидные визуальные баги;
- только релевантные пользовательские сценарии.

Не анализируй весь проект и всю дизайн-систему.
Сначала найди файлы конкретного экрана.
Если нужны браузерные проверки, запусти только нужный локальный сценарий.

Финальный ответ: список найденных проблем по важности, что исправлено, чем проверено.

============================================================
4. CONTINUE IN NEW CHAT
============================================================

Я продолжаю работу в новом чате.

Работай экономно по контексту.

Проект: <путь к проекту>

Сначала прочитай только:
- актуальный current-state/context файл;
- next-work/task файл;
- project-map/README файл.

Не читай историю, архивы и старые planning-документы без необходимости.
Если актуальных context-файлов нет, предложи создать минимальный набор:
- CURRENT_STATE.md;
- NEXT_WORK.md;
- PROJECT_MAP.md.

Моя текущая задача:
<задача>

Если задача слишком широкая, предложи самый дешевый полезный первый шаг.

============================================================
5. BEFORE BIG REFACTOR
============================================================

Перед возможным большим рефакторингом помоги оценить, нужен ли он.

Проект: <путь к проекту>

Проблема:
<что стало неудобно, медленно, дорого или рискованно>

Не начинай рефакторинг сразу.
Сначала:
1. найди минимальные признаки проблемы;
2. оцени стоимость и риск изменения;
3. предложи 2-3 варианта:
   - минимальная правка;
   - средняя переработка;
   - большой рефакторинг;
4. укажи, какой вариант дешевле по контексту и почему.

Код менять только после отдельного подтверждения.
